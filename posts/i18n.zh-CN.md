## 概述

S-Blog 内置完整的多语言 (i18n) 支持，涵盖两个层面：

1. **UI 界面多语言** — 导航栏、按钮、提示文字等均支持中文、英文、日文三种语言
2. **文章内容多语言** — 同一篇文章可以有多个语言版本，读者可按自己的语言偏好阅读

---

## UI 语言切换

> 💡 **试试看：** 点击本页顶部标题旁的语言切换按钮（中 | En | 日），即可实时预览 UI 语言切换效果。

### 语言切换器

页面顶部提供语言切换按钮：**中 | En | 日**

点击即可切换整个站点的 UI 语言。切换后，导航栏、侧边栏、搜索框、相册页面等所有界面元素都会更新为对应语言。

### 支持的语言

| 语言代码 | 语言 | 说明 |
|---------|------|------|
| `en` | English | 默认回退语言 |
| `zh-CN` | 中文 | 简体中文 |
| `ja` | 日本語 | 日语 |

### 语言检测

系统按以下优先级自动检测用户语言：

1. `localStorage` 中保存的上次选择
2. 浏览器语言设置

首次访问时，系统会使用浏览器语言。一旦用户手动切换，选择会被记住。

## 文章多语言

### 文件命名约定

通过文件名后缀来区分语言版本：

```
posts/
├── About.md          # 默认版本（无后缀）
├── About.zh-CN.md    # 中文版本
└── About.ja.md       # 日文版本
```

**规则：**
- 无后缀的文件（如 `About.md`）是默认版本
- 后缀格式为 `.{语言代码}.md`，例如 `.zh-CN.md`、`.ja.md`
- 所有同名文件（slug 相同）会被识别为同一篇文章的不同语言版本

### 示例

假设你想发布一篇关于部署的文章，有中文和英文两个版本：

```
posts/
├── Deployment-Guide.md        # 英文版（默认）
└── Deployment-Guide.zh-CN.md  # 中文版
```

两个文件各自需要完整的 frontmatter：

**`Deployment-Guide.md`:**
```yaml
---
title: Deployment Guide
date: 2025-06-01
tags: [deployment, guide]
categories: [Tutorial]
preview: How to deploy your S-Blog site.
---
```

**`Deployment-Guide.zh-CN.md`:**
```yaml
---
title: 部署指南
date: 2025-06-01
tags: [deployment, guide]
categories: [Tutorial]
preview: 如何部署你的 S-Blog 站点。
---
```

构建后，文章列表中只显示为一篇文章。根据用户当前语言，系统会自动选择对应的标题和摘要来展示。

### 语言回退机制

当用户切换到某个语言，但当前文章没有该语言版本时：

1. 系统显示默认版本的内容
2. 页面顶部出现一条提示，告知用户当前显示的不是其偏好语言的版本

例如，当用户使用日文界面查看一篇只有英文和中文版本的文章时，会看到：
> この記事の日本語版はありません。原文を表示しています。

## 配置

### `config.json` 中的 `language` 字段

```json
{
  "language": "zh-CN"
}
```

此字段设置站点的**默认语言**，影响以下行为：

- **文章列表展示** — 当有本地化的标题和摘要时，优先使用
- **回退判断** — 当用户语言与站点默认语言一致时，不会显示回退提示（因为看到的就是"原版"）

如果不设置此字段，系统默认使用 `en`。

## 开发者参考

### 技术实现

S-Blog 的 i18n 基于 [i18next](https://www.i18next.com/) + [react-i18next](https://react.i18next.com/) 实现：

- **翻译资源** — 内联在 `@s-blog/core` 的 `i18n.ts` 文件中
- **语言检测** — 使用 `i18next-browser-languagedetector`，优先 localStorage 再 navigator
- **内容多语言** — 构建引擎在生成 manifest 时扫描文件名后缀，输出 `availableLanguages` 和 `localizedMeta` 字段

### Manifest 数据结构

构建后的 `manifest.json` 中，多语言文章会包含以下额外字段：

```json
{
  "slug": "About",
  "title": "About This Blog System",
  "availableLanguages": ["zh-CN"],
  "localizedMeta": {
    "zh-CN": {
      "title": "关于本博客系统",
      "summary": "介绍本网站背后的博客系统..."
    }
  }
}
```

### 扩展翻译

如果你 fork 了本项目并希望修改 UI 翻译字符串，翻译资源位于：

```
packages/core/src/i18n.ts
```

每种语言的翻译以嵌套对象组织，包含 `nav`、`common`、`albums`、`post` 等命名空间。
