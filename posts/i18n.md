## Overview

S-Blog provides built-in internationalization (i18n) at two levels:

1. **UI language** — Navigation, buttons, and system text support English, Chinese, and Japanese
2. **Post content** — The same post can have multiple language versions

---

## UI Language Switching

> 💡 **Try it now:** Click the language switcher next to the site title at the top of this page (中 | En | 日) to preview the UI language switching in action.

A language switcher (**中 | En | 日**) is available in the header. Clicking it switches the entire site UI.

The system detects the user's language preference from `localStorage` (previous choice) or browser settings.

## Multi-language Posts

Use filename suffixes to create localized versions of a post:

```
posts/
├── About.md          # Default version
├── About.zh-CN.md    # Chinese version
└── About.ja.md       # Japanese version
```

All files with the same slug are recognized as language variants of one post. The build engine automatically tracks available languages in the manifest.

### Fallback Behavior

When a user's language has no matching post version:
- The default version is displayed
- A notice bar informs the user

## Configuration

Set the default language in `config.json`:

```json
{
  "language": "zh-CN"
}
```

This affects fallback behavior — users browsing in the site's default language won't see fallback notices.
