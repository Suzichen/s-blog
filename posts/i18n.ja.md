## 概要

spage は以下の2つの側面から、完全な多言語 (i18n) サポートを提供しています：

1. **UI の多言語化** — ナビゲーション、ボタン、システムテキストが英語・中国語・日本語に対応
2. **記事コンテンツの多言語化** — 同じ記事に複数の言語バージョンを作成可能。読者は好みの言語で記事を読むことができます。

---

## UI 言語切り替え

> 💡 **試してみましょう：** このページ上部のタイトル横にある言語切り替えボタン（中 | En | 日）をクリックすると、UI 言語切り替えの効果をリアルタイムで確認できます。

### 言語切り替えボタン

ページ上部に言語切り替えボタンがあります：**中 | En | 日**

クリックするとサイト全体の UI 言語が切り替わります。ナビゲーション、サイドバー、検索ボックス、アルバムページなど、すべてのインターフェース要素が対応する言語に更新されます。

### 対応言語

| 言語コード | 言語 | 説明 |
|---------|------|------|
| `en` | English | デフォルトのフォールバック言語 |
| `zh-CN` | 中文 | 簡体字中国語 |
| `ja` | 日本語 | 日本語 |

### 言語の自動検出

システムは以下の優先順位でユーザーの言語を自動検出します：

1. `localStorage` に保存された前回の選択
2. ブラウザの言語設定

初回アクセス時はブラウザの言語設定が使用されます。ユーザーが手動で切り替えると、その選択が記憶されます。

## 記事の多言語対応

### ファイル命名規則

ファイル名のサフィックスで言語バージョンを区別します：

```
posts/
├── About.md          # デフォルトバージョン（サフィックスなし）
├── About.zh-CN.md    # 中国語バージョン
└── About.ja.md       # 日本語バージョン
```

**ルール：**
- サフィックスなしのファイル（例：`About.md`）がデフォルトバージョン
- サフィックスの形式は `.{言語コード}.md`（例：`.zh-CN.md`、`.ja.md`）
- 同じ slug を持つファイルは、同一記事の異なる言語バージョンとして認識されます。

### 例

デプロイに関する記事を英語と日本語で公開したい場合：

```
posts/
├── Deployment-Guide.md        # 英語版（デフォルト）
└── Deployment-Guide.ja.md     # 日本語版
```

各ファイルにはそれぞれ完全な frontmatter が必要です：

**`Deployment-Guide.md`:**
```yaml
---
title: Deployment Guide
date: 2025-06-01
tags: [deployment, guide]
categories: [Tutorial]
preview: How to deploy your blog site.
---
```

**`Deployment-Guide.ja.md`:**
```yaml
---
title: デプロイガイド
date: 2025-06-01
tags: [deployment, guide]
categories: [Tutorial]
preview: ブログサイトをデプロイする方法。
---
```

ビルド後、記事一覧では1つの記事として表示されます。ユーザーの現在の言語に応じて、対応するタイトルと要約が自動的に選択されます。

### 言語フォールバック

ユーザーが特定の言語を選択している状態で、該当記事にその言語のバージョンが存在しない場合：

1. デフォルトバージョンのコンテンツが表示されます
2. ページ上部に、ユーザーの希望する言語バージョンが存在しない旨の通知が表示されます

例えば、中国語インターフェースで英語と日本語バージョンしかない記事を閲覧すると、以下のような通知が表示されます：
> 此文章暂无中文版本，已为您展示作者原文。

## 設定

### `config.json` の `language` フィールド

```json
{
  "language": "ja"
}
```

このフィールドはサイトの**デフォルト言語**を設定し、以下の動作に影響します：

- **記事一覧の表示** — ローカライズされたタイトルと要約がある場合、それを優先的に使用
- **フォールバック判定** — ユーザーの言語がサイトのデフォルト言語と一致する場合、フォールバック通知は表示されない（「オリジナル」を閲覧しているため）

このフィールドを設定しない場合、システムのデフォルトは `en` になります。

## 開発者向け情報

### 技術的な実装

spage の i18n は [i18next](https://www.i18next.com/) + [react-i18next](https://react.i18next.com/) をベースに実装されています：

- **翻訳リソース** — `@spage/core` の `i18n.ts` ファイルにインラインで定義
- **言語検出** — `i18next-browser-languagedetector` を使用、localStorage を優先し次に navigator を参照
- **コンテンツの多言語化** — ビルドエンジンが manifest 生成時にファイル名サフィックスをスキャンし、`availableLanguages` と `localizedMeta` フィールドを出力

### Manifest データ構造

ビルド後の `manifest.json` では、多言語記事に以下の追加フィールドが含まれます：

```json
{
  "slug": "About",
  "title": "About This Blog System",
  "availableLanguages": ["zh-CN", "ja"],
  "localizedMeta": {
    "zh-CN": {
      "title": "关于本博客系统",
      "summary": "介绍本网站背后的博客系统..."
    },
    "ja": {
      "title": "このブログシステムについて",
      "summary": "このサイトを支えるブログシステムの紹介..."
    }
  }
}
```

### 翻訳の拡張

本プロジェクトを fork して UI の翻訳文字列を変更したい場合、翻訳リソースは以下にあります：

```
packages/core/src/i18n.ts
```

各言語の翻訳はネストされたオブジェクトとして整理されており、`nav`、`common`、`albums`、`post` などの名前空間を含みます。

## まとめ

| 機能 | 説明 |
|------|------|
| UI 言語切り替え | 三言語対応、ワンクリック切り替え、選択を自動記憶 |
| 記事の多言語化 | ファイル名サフィックス規則、設定不要 |
| 言語フォールバック | フォールバック表示 + 通知表示 |
| 言語検出 | ブラウザ設定 → localStorage |
| デフォルト言語 | `config.json` の `language` フィールド |
